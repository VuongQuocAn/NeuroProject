## 🛑 1. Critical Issues & Security
- **Login Bypass:** Users can freely navigate to internal pages (e.g., `/patients`, `/history`) without being authenticated by accessing the URLs directly. The Next.js [middleware.ts](file:///d:/Antigravity/NeuroProject/frontend/src/middleware.ts) is likely not protecting these routes.
- **Broken Dashboard Redirect:** The root URL (`/`) incorrectly redirects to `/login` automatically, even if the user is supposedly authenticated. 

## ⚠️ 2. Functionality & Logic Issues
- **Patient Search Failure:** The search bar inside `/patients` does not filter the patient list. The component lacks proper state management (`useState`) or event binding.
- **Dead UI Buttons:**
    - **History Page:** "Xem Báo cáo" (View Report) and "Mở trong trình xem" (Open viewer) buttons have no `onClick` handlers.
    - **Reports Page:** "Chia sẻ" (Share), "In Báo Cáo" (Print), and "Xuất PDF" (Export) are non-functional.
    - **Settings Page:** The non-active tabs show placeholder text, and buttons don't save settings.
- **Language Toggle:** The VN/EN toggle in the sidebar/header is non-functional.

## 🎨 3. UI/UX & Formatting
- **Hydration Mismatch:** Console logs show React hydration errors, which could be caused by mismatched time localization or invalid HTML nesting.
- **Chart Dimension Warnings:** Warnings like `"The width(-1) and height(-1) of chart should be greater than 0"` indicate CSS styling issues on charts rendering before layout constraints are applied.
- **UX Misdirection:** Clicking "Thêm bệnh nhân mới" sends users to the `/upload` page.

---

## Proposed Fixes

1. **Fix Login Bypass:** Ensure [src/middleware.ts](file:///d:/Antigravity/NeuroProject/frontend/src/middleware.ts) checks for a session token and redirects unauthenticated users contextually.
2. **Search Logic in Patients Page:** Update `src/app/(dashboard)/patients/page.tsx` with a robust `searchQuery` state and a filter over the `patientsData` array.
3. **Add Event Handlers:** Inject `onClick={...}` into the buttons across History, Reports, and Settings pages, even if it just triggers a `toast` or `alert('Chức năng đang phát triển')` for now.
4. **Fix Hydration & Charts:** Ensure dates are formatted consistently client-side or use `suppressHydrationWarning`. Provide explicit `minHeight` for chart wrappers.
